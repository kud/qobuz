`@kud/qobuz` is a typed, ESM-only client for the (reverse-engineered) Qobuz v0.2 API. It wraps authentication, search, rich metadata, full favourites and playlist management, deep links into the Qobuz apps, reading the desktop app's now-playing track, and collection analytics from its local library — returning clean camelCase domain types, with zero runtime dependencies. It's the shared core that [`@kud/qobuz-cli`](https://kud.io/projects/qobuz-cli) builds on.

## 🔑 How authentication works

Qobuz has no public API and guards login behind a captcha, so `@kud/qobuz` never
handles passwords. Instead you give it a `user_auth_token` borrowed from a
logged-in browser session, and a pluggable credential store keeps it — in-memory,
the macOS Keychain, or your own implementation. Every request rides on that
token; the library never sees a password.

## 📦 Install

```sh
npm install @kud/qobuz
```
